01

Colors

Our palette is built around a bold purple core with a dark navy anchor. Supplemental colors add energy and are used for data visualization, highlights, and illustrations.

Core Colors

Purple 600 Primary Brand
Black Primary Text & BG

Supplemental Colors

Purple 500 Hover / Active States
Blue 400 Links / Info
Cyan 300 Accent / Highlight
Yellow 300 Warning / Warmth

Usage Guidelines

Do
  • Use Purple 600 as the primary CTA and brand color
  • Use Black for primary text and dark backgrounds
  • Use supplemental colors for data viz and illustrations
  • Maintain sufficient contrast (4.5:1 minimum for text)
Don't
  • Use supplemental colors as primary CTAs
  • Place Cyan or Yellow text on white backgrounds
  • Use more than 3 supplemental colors in one composition
  • Introduce off-palette colors without brand approval

Clear space

04

Voice & Tone

OneSignal speaks like a sharp, technically-fluent teammate who genuinely wants to help you succeed. We're confident without being arrogant, and direct without being cold.

⚡

Sharp

We get to the point. No fluff, no filler. Every word earns its place. We trust our readers to be smart.

🔒

Dependable

We speak with confidence and back it up with data. Promises are kept. Commitments are real.

🙋

Helpful

We're geeky helpers — excited to solve problems. We explain the "why," not just the "how."

🎯

Clear

We avoid jargon unless our audience uses it first. Complex ideas are explained simply, not dumbed down.

🚀

Fast

Our language moves quickly. Short sentences. Active voice. We write the way our product delivers — instantly.

💬

Bold

We take a clear stance. We don't hedge or soften unnecessarily. We believe in what we're building.

In Practice

✗ Not this
"We are pleased to offer a robust, enterprise-grade messaging solution that leverages cutting-edge technology to facilitate user engagement across multiple channels."
✓ This
"Send push notifications, emails, and SMS — all from one dashboard. Set it up in minutes."
✗ Not this
"Your message may have potentially experienced some delivery issues at this time."
✓ This
"3 messages failed to deliver. Check your API keys and retry."
✗ Not this
"Utilize our sophisticated segmentation capabilities to optimally target your audience demographics."
✓ This
"Segment by behavior, location, or custom data. Send the right message to the right person."

Word List

Use
  • Send / Deliver
  • Real-time
  • Segment
  • Users / Subscribers
  • Set up (not "onboard")
  • Dashboard
  • Notification
  • Scale
Avoid
  • Leverage (use "use")
  • Utilize (use "use")
  • Robust (use specific adjectives)
  • Synergy / Ecosystem
  • Journey (unless literal)
  • Revolutionize
  • Cutting-edge / State-of-the-art
  • Seamless (show it, don't say it)

Iconography

We use the Streamline Regular icon set — a clean, consistent line-style library that pairs well with Epilogue's sharp geometry. Icons communicate function, not decoration.

Library Streamline Regular
Style 1.5px stroke · Line / Outline
Sizes 16px · 20px · 24px · 32px · 48px
Format SVG preferred · PNG @2x fallback
Source streamlinehq.com ↗

Icon Usage Guidelines

Do
  • Use icons at their intended pixel sizes (16, 20, 24, 32, 48px)
  • Keep stroke weight consistent — always 1.5px in Streamline Regular
  • Color icons using brand palette tokens only
  • Pair icons with text labels in UI contexts
  • Use a single icon style throughout a product or document
Don't
  • Mix Streamline Regular with filled or other icon families
  • Scale icons to non-standard sizes (e.g. 18px, 22px)
  • Use icons without sufficient color contrast
  • Use icons alone as the only means to convey critical information
  • Add drop shadows or visual effects to icons

06

Copywriting

OneSignal's microcopy follows Material Design content principles and our internal style guide. Every word in our UI should feel sharp, helpful, and human — never corporate or vague.

Core Principles

1

Be concise

Write only what users need. Remove filler words. If a label works without a supporting sentence, skip the sentence.

2

Be specific

Avoid vague copy. "Error occurred" tells users nothing. "Couldn't connect — check your API key." gives them a path forward.

3

Use active voice

"Send your message" not "Your message will be sent." Active voice is faster to read and feels more direct.

4

Front-load meaning

The most important word goes first. Users scan — don't bury the key information at the end of a sentence.

5

Match the moment

Tone shifts with context. Errors should be calm and helpful, not alarming. Success moments can be warm. Onboarding can be encouraging.

6

Avoid jargon

Write for a developer who's new to OneSignal, not for an internal team member. If you must use a technical term, define it nearby.

UI Element Guidelines

Buttons & CTAs

Use verb + noun format. The action should be clear from the label alone — users shouldn't need surrounding context to understand what a button does.

✗ Submit
✗ OK
✓ Send notification
✓ Create segment
✓ Save changes
Error Messages

State what went wrong, then tell users what to do. Never blame the user. Avoid technical error codes in visible UI — log them for support, show a human explanation instead.

✗ Error 403: Forbidden
✗ Something went wrong
✓ You don't have permission to access this. Contact your workspace admin.
Empty States

Empty states are an opportunity, not dead space. Explain what belongs here, and give users a clear action to fill it. Use a friendly, encouraging tone.

✗ No data
✓ No segments yet. Create your first segment to target the right users.
Placeholder Text

Placeholders should demonstrate the expected format — not repeat the label. They disappear on focus, so don't put critical information in them.

✗ Enter your app name
✓ e.g. My iOS App
Tooltips & Helper Text

Be short — tooltips should fit in one line where possible. Don't explain what a user can already see. Explain the "why" or provide context that isn't obvious from the label.

✗ This is the delivery rate for your notification
✓ % of targeted users who received this notification
Confirmation Dialogs

The title should state the action, not ask a yes/no question. The body explains the consequence. The primary button confirms the action using the same verb from the title.

✗ Title: "Are you sure?" · Button: "Yes"
✓ Title: "Delete segment?" · Body: "This can't be undone." · Button: "Delete segment"

Capitalization & Punctuation

Sentence case everywhere

Use sentence case for all UI text — headings, buttons, labels, tooltips. The only exception is proper nouns and product names.

✗ Create A New Segment
✓ Create a new segment

Skip terminal periods

Don't end labels, buttons, headings, or tooltips with a period. Use periods only in full body paragraphs with multiple sentences.

✗ Send notification.
✓ Send notification

Ampersands in UI

Use "&" only in navigation labels or tight UI where space is constrained. In body copy and error messages, write "and".

Numbers

Use numerals (1, 2, 3) in all UI contexts — not written out. Spell out numbers only at the very start of a sentence.

✗ Three messages failed
✓ 3 messages failed
